## Trigger

Load this skill whenever tasked with analyzing or clearing mutation testing survivors (`mutmut`), configuring `[tool.mutmut]` in `pyproject.toml`, or running incremental mutation-clearing loops across repository modules.

## Reference

### The 4-Archetype Triage Taxonomy

Every surviving mutant generated by `mutmut` maps to exactly one of four archetypes:

| Archetype | Mutation Shape | Root Cause | Required Action |
| :--- | :--- | :--- | :--- |
| **I. Unasserted Branch / Fallback** | Mutating return values or state inside error handlers or fallback branches (`if x is None: return ...`). | Unit tests exercise helper functions directly but never call the top-level API on inputs triggering the branch. | **Add Branch Test:** Add a unit test calling the top-level gate/API on inputs that trigger the fallback branch, asserting the full return dataclass. |
| **II. Dropped Audit / Telemetry Field** | Setting dataclass attributes or event fields to `None` instead of passing the preserved object (`canonical_target=None`). | Tests assert the boolean decision (`allow=False`) but neglect auxiliary audit telemetry fields. | **Assert Full Dataclass:** Pin exact dataclass instances (`result == ExpectedDataclass(...)`) on boundary denials to protect audit logging invariants. |
| **III. Contract / Security String** | Altering reason strings, diagnostic explanations, or operator-facing security messages. | Loose substring checks (`assert "unresolved" in result.reason`) permit corrupted or missing security context. | **Pin Contract:** Pin exact multiline reason strings where the string is an operator-facing explanation from a security boundary gate. |
| **IV. API Default / Parameter Drift** | Flipping boolean parameters or API flags (`strict=False` $\rightarrow$ `strict=None` / `True`). | Tests only supply "happy path" inputs where defaults and explicit overrides behave identically. | **Test Edge Input or Spy:** Pass input where parameter variance alters observable behavior, or use a lightweight spy (`monkeypatch.setattr`). |

### Accepted Equivalent Mutants Ledger Criteria

Per project rules, **never write mock-heavy tests that mirror implementation lines** just to kill a mutant. If a mutant alters unobservable internal behavior or represents Python standard library invariants, annotate the source line with `# pragma: no mutate` and log it under the workplan ledger with one of these canonical justifications:

- **Dead Defensive Guard:** Data invariants guaranteed by upstream constructors (e.g. `pathlib.Path.parts` stripping empty/dot components).
- **Mathematical / Logical Redundancy:** String stripping or comparison logic already covered by boundary checks on prior lines (e.g. `a == b` check preceding `rstrip("/") == rstrip("/")`).
- **Standard Library Default:** Keyword argument matches standard library default (e.g. `shlex.split(..., posix=True)`).

## Decision points

1. **Test Selection Check (`pyproject.toml`):** Before hunting mutants in any module listed in `source_paths`, verify that its unit test file is registered in `pytest_add_cli_args_test_selection`. If missing, add it before running `mutmut`.
2. **Spy Isolation Rule:** When mocking or spying on universal built-ins (e.g. `Path.resolve`), ensure the spy filter records arguments without asserting inside the wrapper. Mutmut's internal runner (`record_trampoline_hit`) executes host functions during mutation generation and will crash if test spies assert against mutmut's own file operations.
3. **Existing Test Protection:** Modifying existing tests during a bug fix requires a `TEST-EDITS: <path> — <reason>` declaration in the PR description per the `pr-creation` skill.

## Output format

When proposing or committing mutation fixes, structure the summary into two explicit lists:

```text
KILLED VIA EXACT ASSERTIONS (<count>):
- <mutant-id>: <brief explanation of test added>

ACCEPTED AS EQUIVALENT / UNOBSERVABLE (<count>):
- <mutant-id> (`<file>:<line>`): <rationale matching Reference criteria>
```

## What the hook validates

- `mutmut run` runs cleanly against the configured scope in `pyproject.toml` and produces `mutants/mutmut-cicd-stats.json`.
- `check_protected_paths.py` verifies that any edits to protected test suites are declared properly.
- `just check` verifies that no style, linting, complexity (`C901`), or type checking (`mypy strict`) regressions were introduced.

## Escalation

If an apparent equivalent mutant cannot be proven unobservable or if `# pragma: no mutate` annotations would exceed 10% of a module's executable lines, stop and perform a structural refactor to simplify the control flow rather than suppressing mutation testing.
