# Инструкции по развёртыванию и эксплуатации First-Agent

Это «единая точка входа» для человека, который **разворачивает** или
**администрирует** First-Agent на выделенном сервере (AIO — «всё в одном»).
Документы разделены по сценарию использования — выберите свой:

## С чего начать

| Я хочу… | Открыть |
|---------|---------|
| **Развернуть с нуля** (новый сервер: BIOS → Ubuntu → Docker → Tailscale → первый запуск) — делается **один раз** | [`01-install.md`](./01-install.md) |
| **Обслуживать работающий деплой** (обновление, пересборка, бэкапы, перезапуск, диагностика) — **повседневно** | [`02-operations.md`](./02-operations.md) |
| **Усилить защиту SSH** (firewall, fail2ban, Tailscale ACL) | [`../../scripts/ssh-tailscale/README.md`](../../scripts/ssh-tailscale/README.md) |

## Как это устроено (принцип разделения)

- **`01-install.md`** отвечает за **однократный** путь «от чистого железа до
  первого запущенного агента». Сюда не входят повторяющиеся операции.
- **`02-operations.md`** отвечает за **всё повторяющееся**: обновления,
  пересборку, бэкапы, управление сервисом, восстановление и диагностику.
- Логика усиления SSH живёт в **одном** месте —
  [`scripts/ssh-tailscale/README.md`](../../scripts/ssh-tailscale/README.md);
  обе инструкции ссылаются на него, не дублируя шаги.

Так каждый документ остаётся в пределах своего объёма и читается линейно под
конкретную задачу, а пересечений между файлами нет.
