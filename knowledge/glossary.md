# Glossary

Short definitions of terms used in First-Agent and this wiki.

| Term | Description  |
|---|---|
| **ACI** | Agent–Computer Interface. The minimal surface through which an agent interacts with a computational environment (tools, files, shell). See [`research/cutting-edge-agent-research-radar-2026-05.md`](../knowledge/research/cutting-edge-agent-research-radar-2026-05.md). |
| **ADR** | Architecture Decision Record. Canonical location: [`knowledge/adr/`](../knowledge/adr/); process: [`knowledge/adr/README.md`](../knowledge/adr/README.md). Cheat sheet: [`knowledge/adr/DIGEST.md`](../knowledge/adr/DIGEST.md). |
| **ADR DIGEST** | One-paragraph cheat-sheet per accepted ADR. Cheap-read overlay; per-ADR file authoritative. Updated in the same PR as any amendment ([AGENTS.md rule #9](../AGENTS.md#pr-checklist)). Canonical: [`knowledge/adr/DIGEST.md`](../knowledge/adr/DIGEST.md). |
| **ADVISORY** | Authoring-guardrail severity that is visible in output but does **not** fail CI (exit 0). Must carry an `expires_on` date or auto-escalate; promotes to *HARD-BLOCK* only after a measured false-positive rate < 1 % on `fp-corpus/`. See [ADR-11 §ADR-11-I2](../knowledge/adr/ADR-11-authoring-guardrails.md#adr-11-i2--severity-lifecycle-as-a-false-positive-budget). |
| **Agent** | A program that uses an LLM to select a sequence of actions (usually via tool calls) in pursuit of a goal. |
| **Aperant** | OSS reference TypeScript desktop coding-agent. Cited in [`research/gortex-aperant-inspiration-2026-05.md`](../knowledge/research/gortex-aperant-inspiration-2026-05.md) as primary source for `record_gotcha` / `record_discovery` (R-8), pause-handler sentinel (R-25), bash-validator + path-containment (R-20). FA borrows shape, not code; TS → Python ports preserve atomic-rename idiom + empirically-validated intervals. |
| **Ara protocol** | Four-layer multi-agent research protocol from arXiv:2604.24658v2 (Orchestra Research): Live Research Manager / Compiler / Seal. First-Agent borrows only the `/trace/` *Exploration log* idea (R-1 of the cross-reference); the rest is out of scope for v0.1. See [`research/ara-protocol-cross-reference-2026-05.md`](../knowledge/research/ara-protocol-cross-reference-2026-05.md). |
| **Ask Devin** | Devin feature: task scoping, codebase search and automatic construction of a high-context prompt prior to opening a session. |
| **Authoring-time guardrail** | Deterministic repository check that fires while writing/reviewing code — before runtime behaviour matters — under the *LLM as Untrusted Compiler* threat model. Enforced by the *Two-tier TCB*. See [ADR-11](../knowledge/adr/ADR-11-authoring-guardrails.md). |
| **Auto-Fix** | Devin Review feature: automatic responses to review comments and CI failures. |
| **Axis (project)** | High level project axis, synonym for *Pillar* as defined in [`project-overview.md` §1.1](../knowledge/project-overview.md#11-четыре-столпа-цели-project-goal--four-pillars). Used in phrases such as "follow project axis". |
| **Axis (PR-checklist A/B/C)** | Evaluation criterion in §0 Decision Briefing per [AGENTS.md PR Checklist rule #8](../AGENTS.md#pr-checklist): (A) reduces session-start noise; (B) helps LLM locate context; (C) advances the selected `goal_lens`. (A) and (B) are stable axes for all notes; (C) is per-session. |
| **Bash sandbox / `bash_gate`** | Layered bash command validator (R-20 in [`research/borrow-roadmap-2026-05.md`](../knowledge/research/borrow-roadmap-2026-05.md)): Layer 1 pattern classifier (zero-latency `read-only` / `git-write` / `package-manager-install` / `dangerous` bucketing); Layer 2 per-command validators for `rm` / `chmod` / `git` / `pkill` / `psql`; Layer 3 path-containment (resolves symlinks, blocks `..` traversal). v0.2 [ADR-6](../knowledge/adr/ADR-6-tool-sandbox-allow-list.md) sandbox upgrade. Default-deny with a per-classifier exception list. |
| **BM25 / FTS5** | The v0.1 read-side ranker. SQLite FTS5 in external-content mode with tokeniser `unicode61 remove_diacritics 2` + porter stemmer; default BM25 weights. **No vector layer in v0.1**; v0.2 ADR slot reserved for `sqlite-vec` or a separate `embeddings.sqlite`. See [ADR-4](../knowledge/adr/ADR-4-storage-backend.md). |
| **Capability flag** | One of five Kronos-style boolean flags (R-21 in [`research/borrow-roadmap-2026-05.md`](../knowledge/research/borrow-roadmap-2026-05.md)) that gate runtime features: `ENABLE_DYNAMIC_TOOLS`, `REQUIRE_DYNAMIC_TOOL_SANDBOX`, `ENABLE_MCP_GATEWAY_MANAGEMENT`, `ENABLE_DYNAMIC_MCP_SERVERS`, `ENABLE_SERVER_OPS`. All default `False`; capability enables via config file (audit-able), not runtime decision. Land in v0.2 [ADR-6](../knowledge/adr/ADR-6-tool-sandbox-allow-list.md) amendment. |
| **Catch-corpus** | Fixture diffs from historical authoring omissions (F-1..F-10) that rules MUST flag — the true-positive baseline a rule must catch before promotion. Lives at `catch-corpus/`. See [ADR-11 §Verification](../knowledge/adr/ADR-11-authoring-guardrails.md#verification). |
| **Chain of custody** | Provenance rule: when citing a number, date, name, or quote, fetch it from the primary source listed in the note's `source:` frontmatter — **not** from a summary note. Documented in [AGENTS.md §Query Routing](../AGENTS.md#query-routing); rationale lives in [`research/llm-wiki-critique.md`](../knowledge/research/llm-wiki-critique.md). |
| **Cost guardian** | R-45 middleware shipped 2026-05-21 as `src/fa/observability/cost_guardian.py`. Single `GuardMiddleware` that attaches to both `BEFORE_TOOL_EXEC` (gates when accumulated USD exceeds `RuntimeLimits.cost_budget_usd`) and `AFTER_TOOL_EXEC` (parses `cost=tokens_in=<int>,tokens_out=<int>,usd=<float>` artifacts via `default_cost_extractor`, accumulates `CostRollup`, writes `kind="cost_observation"` audit rows). `cost_budget_usd` is tri-mode — `None` unbounded (default), `0.0` observe-only, `> 0` hard cap. Dormant on M-1 baseline tools; wakes when the T-2 LLM driver emits the cost artifact. Documented in [ADR-7 §Sub-amendment 2026-05-21](../knowledge/adr/ADR-7-inner-loop-tool-registry.md#sub-amendment-2026-05-21--r-45-cost-guardian--cost_observation-event-kind). |
| **Decision Briefing (§0)** | Mandatory first section of research notes produced via [`prompts/research-briefing.md`](../knowledge/prompts/research-briefing.md). Eight-field R-N block + 7-column summary table; full schema in [AGENTS.md PR Checklist rule #8](../AGENTS.md#pr-checklist). Lets agents read §0 alone (~150–250 lines) without loading the deep dive. |
| **Devin Review** | Devin's automatic PR reviewer. |
| **DPC** | Disentinel Personal Companion — OSS reference Python LLM-agent stack with mature ADR / lessons-learned log. Cited in [`research/dpc-messenger-inspiration-2026-05.md`](../knowledge/research/dpc-messenger-inspiration-2026-05.md) and [AGENTS.md §Cross-project anti-patterns](../AGENTS.md#cross-project-anti-patterns) for four empirical anchors (ADR-015 evolution-worker removal; ADR-005 P18 scope-only estimation; ADR-021 Lesson 4 write-only-subsystem rule; AP8 Prior-Art mapping). |
| **DSV** | Deterministic State Verification. Post-tool gate (R-5 in [`research/borrow-roadmap-2026-05.md`](../knowledge/research/borrow-roadmap-2026-05.md)) that compares the LLM-claimed success of a tool call against the actual events emitted during execution and overrides the success flag on mismatch. Implementation lives at `src/fa/verifier/` (Wave 0 standalone module; integrates into the [ADR-7](../knowledge/adr/ADR-7-inner-loop-tool-registry.md) hook pipeline once the [HookRegistry](#hookregistry) lands). |
| **Draft PR** | A Pull Request in draft state; CI runs, but it is not marked as ready-for-review. |
| **Eval / Eval suite** | Reproducible set of inputs + expected behaviour used to measure agent quality between changes. |
| **Exploration log** | Telegraphic markdown overlay at [`knowledge/trace/exploration_log.md`](../knowledge/trace/exploration_log.md). One `## Q-N` block per ADR with `Chosen:` + each `Rejected:` carrying `Reason:` (why rejected) + `Lesson:` (what new evidence re-opens it). Schema: [`knowledge/README.md §trace/`](../knowledge/README.md#trace--exploration-log). Converted from YAML DAG 2026-05-10 per Tsinghua NLAH (origin: [`research/ara-protocol-cross-reference-2026-05.md` §9 R-1](../knowledge/research/ara-protocol-cross-reference-2026-05.md)). |
| **Family extractor** | R-19 helper shipped 2026-05-21 as `src/fa/roles.py` — pure-function module that maps an LLM model slug to a training-distribution family (member of the 10-entry `KNOWN_FAMILIES` frozenset: anthropic, openai, google, qwen, deepseek, glm, kimi, mimo, nemotron, llama). `extract_family(slug, *, override=None) -> str` runs an ordered regex table; raises `FamilyExtractionError` on ambiguous slugs so the caller must pass an explicit `family:` override from `~/.fa/models.yaml`. Used by the loader's `check_eval_disjoint` call to enforce the [ADR-2 §Amendment 2026-05-20](../knowledge/adr/ADR-2-llm-tiering.md#amendment-2026-05-20--eval-role-family-disjoint--primary-source-citation) rule that the Eval role must not share a family with Planner or Coder (Cornell P-1 + Simula P-2 evidence). Hook-layer call site already lives in [ADR-7 §Amendment 2026-05-20](../knowledge/adr/ADR-7-inner-loop-tool-registry.md#amendment-2026-05-20--retry-budget-invariant-intra-role-t10-llm-using-hook-family-disjoint-rule) rule 4. |
| **Feedback loop** | The cycle `action → observation → reflection → next action` — the core pattern of a reliable agent. |
| **Filesystem-canon** | Project convention that durable agent memory lives in ordinary repository files (Markdown / JSON / YAML) instead of opaque service state. R-8 uses `knowledge/trace/gotchas.md` + `knowledge/trace/codebase_map.json` as canonical learning artifacts. |
| **FP-corpus** | Diffs from recent green commits that rules MUST NOT flag; used to measure false-positive rate (gate < 1 %) before an *ADVISORY* rule promotes to *HARD-BLOCK*. Lives at `fp-corpus/`. See [ADR-11 §Verification](../knowledge/adr/ADR-11-authoring-guardrails.md#verification). |
| **goal_lens** | Frontmatter v2 field; one-sentence research goal, elicited at session start (Stage 1) per [`prompts/research-briefing.md`](../knowledge/prompts/research-briefing.md). Mandatory for notes produced by the research-briefing workflow; optional for all others. Allows the agent to filter the corpus against the current task without loading notes. See also *Lens*. |
| **Gortex** | OSS reference Go static-analysis / repo-audit toolchain. Cited in [`research/gortex-aperant-inspiration-2026-05.md`](../knowledge/research/gortex-aperant-inspiration-2026-05.md) as primary source for the tokens classifier + buildSuggestions output formatter + `DefaultConfigPaths` discovery (R-13), workspace resolution rule (R-14), full bloat / staleness audit subsystem (R-15, deferred). FA borrows shape; Go → Python ports. |
| **Golden set** | A small, stable, manually annotated set of inputs used for regression evaluation. |
| **HARD-BLOCK** | Authoring-guardrail severity that fails CI (exit 1). Used only for deterministic, low-noise rules; *I-FROZEN* parity mismatches are HARD-BLOCK. See [ADR-11 §ADR-11-I2](../knowledge/adr/ADR-11-authoring-guardrails.md#adr-11-i2--severity-lifecycle-as-a-false-positive-budget). |
| **Harness** | Control layer around an LLM: loop / orchestration, prompts, tool registry, retrieval pipeline, sandbox. This is **not** ML or the model itself; this is everything an agent needs in between LLM calls. Pillar 3 of the project is to build the most token/tool-call efficient open-source harness for UC1+UC3. |
| **HookRegistry** | Registry-driven middleware chain proposed in future [ADR-8](../knowledge/adr/) (R-1 in [`research/borrow-roadmap-2026-05.md`](../knowledge/research/borrow-roadmap-2026-05.md)). Two middleware kinds: `GuardMiddleware` (may deny / modify; pre-tool / pre-run) and `ObserverMiddleware` (read-only; post-tool / on-event). Substrate for Wave 2 work: `LoopGuard` (R-2), failure-classifier dispatcher (R-3), pre-tool blocker (R-4), PII redaction walker (R-22). Doc-first; runtime is Phase-M scope. |
| **Hook** | Pre/post-tool extension point formalised by [ADR-7](../knowledge/adr/ADR-7-inner-loop-tool-registry.md) §8: `pre_tool` (validation / sandbox checks, may `deny` or `modify_params` — mutation forces JSON-Schema + sandbox re-validation) and `post_tool` (audit / artifact normalisation). v0.1 ships `SandboxHook` ([ADR-6](../knowledge/adr/ADR-6-tool-sandbox-allow-list.md)) + optional `ApprovalHook` + `AuditHook`. `pre_run` / `post_run` / `on_event` deferred to v0.2. |
| **I-FROZEN** | Marker for a guarded block whose edits must pass a checker/generator: `# I-FROZEN: source=<path#anchor> checker=<path-or-rule-code>`. A mismatch on deterministic regeneration is a *HARD-BLOCK* (ADR-11-I3). See [ADR-11](../knowledge/adr/ADR-11-authoring-guardrails.md#adr-11-i3-i-frozen--generatedmirrored-parity). |
| **INFO (severity)** | Authoring-guardrail severity for informational findings; never blocks CI. Lowest rung of the `HARD-BLOCK / ADVISORY / INFO` ladder. See [ADR-11 §ADR-11-I2](../knowledge/adr/ADR-11-authoring-guardrails.md#adr-11-i2--severity-lifecycle-as-a-false-positive-budget). |
| **Inner-loop** | The Coder ↔ tools cycle: thought → tool-call → observation → next thought. Pinned by [ADR-7](../knowledge/adr/ADR-7-inner-loop-tool-registry.md): MCP-shaped `ToolSpec` / `ToolResult` registry, three-tier tool disclosure, JSON-Schema input validation, two edit-shapes (string-replace default + unified-diff feature-flagged), `pre_tool` / `post_tool` hook pipeline, raw `events.jsonl` trace separated from `hot.md` summary, and a static layered prompt frozen at session start. Inputs: [`research/efficient-llm-agent-harness-2026-05.md` §10](../knowledge/research/efficient-llm-agent-harness-2026-05.md), [`research/how-to-build-an-agent-ampcode-2026-04.md`](../knowledge/research/how-to-build-an-agent-ampcode-2026-04.md). |
| **Knowledge note** | A short trigger note that Devin automatically pulls into all future sessions. |
| **Kronos** | OSS reference «agent OS» implementing identity-preserving compaction (R-9), 5-flag capability opt-in (R-21), recursive PII walker (R-22, deferred). Cited in [`research/kronos-agent-os-inspiration-2026-05.md`](../knowledge/research/kronos-agent-os-inspiration-2026-05.md). FA borrows shape; Python → Python ports adopt the file-system canon idiom but adapt to FA's Markdown-as-canon stance. |
| **L0 / L1 / L2 (memory tiers)** | Progressive-disclosure retrieval tiers: **L0** = filename/title/tag grep; **L1** = SQLite FTS5 BM25 over chunks; **L2** = vector / graph (deferred to v0.2). v0.1 ships L0+L1 only per [ADR-3](../knowledge/adr/ADR-3-memory-architecture-variant.md) + [ADR-4](../knowledge/adr/ADR-4-storage-backend.md). L2 is additive, not a migration. |
| **Lens** | See *goal_lens*. Used in phrases such as "follow project axis and lens goal" — almost always refers to the `goal_lens:` frontmatter of a note or session. |
| **Level 0 kernel / Level 1 rule (authoring TCB)** | Two-tier authoring-check split (ADR-11): **Level 0** = frozen, stdlib-only, offline kernel that parses the manifest, enumerates+hashes the snapshot, dispatches an allowlist, and emits sorted diagnostics; **Level 1** = the allowlisted semantic rule packs it dispatches (stdlib `ast` / regex text markers / hashing). See [ADR-11](../knowledge/adr/ADR-11-authoring-guardrails.md#two-tier-tcb-shape-blueprint-91-96). |
| **LLM agent** | See *Agent*. |
| **LLM as Untrusted Compiler** | ADR-11 threat model: the LLM author can edit the guardrails in the same patch as the code they constrain, so authoring checks must sit in a frozen, protected-path-governed TCB the author cannot silently weaken. See [ADR-11 §Context](../knowledge/adr/ADR-11-authoring-guardrails.md#threat-model-llm-as-untrusted-compiler). |
| **Managed Devin** | A child Devin session spawned by another, usually for parallelism. |
| **MCP** | Model Context Protocol — a JSON-RPC shaped contract between an MCP host (agent) and MCP server (source of tools / resources / prompts). FA v0.1 implements a **convention** (in-process dispatcher mirrors JSON-RPC), not a dependency — see [ADR-2 amendment 2026-05-01](../knowledge/adr/ADR-2-llm-tiering.md#amendment-2026-05-01--mcp-forward-compat-tool-shape-convention). In Devin context, also the mechanism for connecting Devin to external systems. |
| **Mechanical Wiki** | v0.1 memory architecture variant ([ADR-3](../knowledge/adr/ADR-3-memory-architecture-variant.md)): filesystem-canonical Markdown + YAML frontmatter, deterministic chunker, SQLite FTS5 read-side. **No embeddings, no graph, no Mem0.** v0.2 hooks scaffolded under `src/fa/memory/volatile/` but empty — additive upgrade, not migration. |
| **Minimalism-first** | Project principle ([`project-overview.md` §1.2](../knowledge/project-overview.md#12-enforceable-principle--minimalism-first)): every proposed harness component passes a 4-question test before addition (research evidence; precedent for removal; replacement capability; deterministic Python function instead of LLM call). Prevention strategy for greenfield projects. Contrast: *Subtraction-first*. |
| **Module** | A self contained unit of agent code located at `src/<name>/` with its own tests and README. |
| **NLAH** | Natural-Language Agent Harness — an externalised, editable natural language artifact describing harness behaviour; runtime is the Intelligent Harness Runtime (IHR). Source: Tsinghua paper `arXiv:2603.25723`. First-Agent already implements the text half of NLAH (AGENTS.md + ADRs + research notes); ADR-7 is the bridge to the code half. See [`research/efficient-llm-agent-harness-2026-05.md`](../knowledge/research/efficient-llm-agent-harness-2026-05.md). |
| **Phase R / S / M** | The project's build lifecycle (canonical definition — folded here 2026-05-29 when the former `docs/workflow.md` was retired). **Phase R — Research:** exit with a reviewed written plan — problem stated + accepted, a reviewed PRD, ≥1 ADR fixing each «big decision» (framework / orchestration style / model), and any unresolved questions explicitly marked *deferred*. **Phase S — Scaffolding (complete):** stand up the feedback loop before the first module — typed Python 3.11+ + `pyproject.toml`, `ruff` lint/format, `mypy --strict`, `pytest`, green CI on `main`, `Makefile` (`make lint/typecheck/test/check`), `.pre-commit-config.yaml`; exit when `make test` returns 0, CI is green, pre-commit works, and a «how to run checks» note exists. **Phase M — Module creation (current, iterative):** per module — draft an ADR for any new decision, scaffold via the T2 prompt → draft PR, human review → request-changes/merge, capture new gotchas in Knowledge notes, and keep the routing surface (`knowledge/llms.txt`) current. See also *R-S-M*. |
| **Pillar** | One of 4 project goal pillars defined in [`project-overview.md` §1.1](../knowledge/project-overview.md#11-четыре-столпа-цели-project-goal--four-pillars): (1) research-backed implementation-first reference; (2) pragmatic single-user product; (3) most token/tool-call efficient open-source harness; (4) iteration via measurement. Synonym *Project axis*. |
| **Playbook** | A reusable multi-step recipe that Devin can execute on demand. |
| **PRD** | Product Requirements Document. For First-Agent this is a short markdown file under `docs/prd/`. |
| **Pause sentinel** | Filesystem-canonical pause/resume primitive (R-25 in [`research/borrow-roadmap-2026-05.md`](../knowledge/research/borrow-roadmap-2026-05.md)): orchestrator writes a `RATE_LIMIT_PAUSE` (with reset timestamp) or `AUTH_PAUSE` file when the LLM tier returns HTTP 429 / 401; a human or frontend writes a `RESUME` file to unblock. Empirically-validated intervals: `MAX_RATE_LIMIT_WAIT_MS = 7_200_000` (2h), `RATE_LIMIT_CHECK_INTERVAL_MS = 30_000` (30s), `AUTH_RESUME_MAX_WAIT_MS = 86_400_000` (24h), `AUTH_RESUME_CHECK_INTERVAL_MS = 10_000` (10s). |
| **Prompt** | An instruction sent to an LLM (or Devin). Reusable prompts are stored in `knowledge/prompts/`. |
| **Provenance frontmatter** | Mandatory `source:` + `compiled:` (+ `chain_of_custody:` when citing facts) on every research note. v2 optional fields: `tier`, `links`, `mentions`, `confidence`, `goal_lens`, `topic`. Schema in [`knowledge/README.md` §Provenance-frontmatter](../knowledge/README.md). |
| **R-N / Q-N** | Notation in research notes and the [exploration log](../knowledge/trace/exploration_log.md): `R-N` = numbered recommendation (TAKE / SKIP / DEFER / UNCERTAIN-ASK verdict in §0 Decision Briefing); `Q-N` = open question (e.g. `Q-1`, `Q-2`); alternative branches per question are listed inline under **Rejected:** bullets in the log; `Q-N amendment YYYY-MM-DD` = follow-up node attached to the original question, with a **Coupling:** field pointing back. |
| **R-S-M** | Lifecycle: Research → Scaffolding → Module. See *Phase R / S / M*. |
| **ReAct** | Reasoning + Acting loop — the ubiquitous agent orchestration pattern: thought → action → thought. |
| **Reflexion / Critic / Reflector** | Inner-loop role that reviews its own output and proposes a retry. **v0.1 inner-loop has no Critic / Reflector role**; failure handling is intra-role retry only per [ADR-2 amendment 2026-04-29](../knowledge/adr/ADR-2-llm-tiering.md#amendment-2026-04-29--tool_protocol-field--native-by-default-v01-inner-loop-without-critic). Reconsider only if eval shows weaker models miss self-correctable errors. |
| **Scheduled session** | A cron-like recurring Devin session — e.g. weekly eval run. |
| **Seam (authoring)** | Declared boundary of intended edits for an authoring session (`.fa/session.toml`); the first enforceable seam rule is *staged paths ⊆ declared seam* (ADR-11-I8 / I-BOOT). See [ADR-11](../knowledge/adr/ADR-11-authoring-guardrails.md#adr-11-i8-i-boot--session-seam-and-bootstrap-read-set). |
| **Self-evolving** | Agent writes its own `SKILL.md` files at session end (planned location: `~/.fa/skills/` or `knowledge/skills/`). v0.1 commitment per Pillar 4 ([`project-overview.md` §1.1](../knowledge/project-overview.md#11-четыре-столпа-цели-project-goal--four-pillars)); design in TBD ADR-8. Distinct from *Reflexion* (per-step vs per-session). |
| **Session Insights** | Post-session analytics: timeline, cost, suggestions for prompt improvement. |
| **Skill** | A `SKILL.md` file in the repository — a procedure that Devin knows how to execute. v0.1 commitment: agent capability to write its own skills under `~/.fa/skills/` or `knowledge/skills/` (Pillar 4 foundation; ADR-8 TBD). |
| **Soviet-code** | OSS reference npm-published agent harness (`Disentinel/soviet-code`, v1.964.0). Cited in [`research/soviet-code-inspiration-2026-05.md`](../knowledge/research/soviet-code-inspiration-2026-05.md) as primary source for declarative per-role tool whitelist (B-NEW-1, see [ADR-7 §Amendment 2026-05-13](../knowledge/adr/ADR-7-inner-loop-tool-registry.md)) and Phase-M runner pattern (B-NEW-3, deferred to Wave 3). 9-dept profile granularity rejected for v0.1 as over-engineered. |
| **Subtraction-first** | Design rule (Anthropic): every harness component encodes a model-limitation assumption; assumptions expire as models improve → remove the component if measurements show it is no longer effective. **Retrofit** strategy for legacy harnesses. FA selected *Minimalism-first* (prevention) over subtraction. |
| **Symmetric reading** | Pre-flight rule ([AGENTS.md §Pre-flight Step 3](../AGENTS.md#pre-flight-checklist)): before citing a research note, run `grep -ril "<key-term>" knowledge/research/` and read every match. Cite all or explain skips in one sentence. |
| **TAKE / SKIP / DEFER / UNCERTAIN-ASK** | Verdict tokens used in §0 Decision Briefing of every cross-reference research note. **TAKE** = act on the recommendation in v0.1; **SKIP** = reject with reason; **DEFER** = revisit in v0.2 or post-UC5; **UNCERTAIN-ASK** = block on the user with a 3–4-option `user_question` before opening the PR. Per [`prompts/research-briefing.md` §Decision Briefing format](../knowledge/prompts/research-briefing.md). |
| **Tool call** | A structured request from an LLM to invoke a named tool with typed arguments. Native vs prompt-only — see [ADR-2 amendment 2026-04-29](../knowledge/adr/ADR-2-llm-tiering.md#amendment-2026-04-29--tool_protocol-field--native-by-default-v01-inner-loop-without-critic). |
| **Tool Registry** | The collection of `ToolSpec` entries (name / description / input_schema / permission / handler / tags / output_schema?) the Coder LLM is allowed to invoke. Pinned by [ADR-7](../knowledge/adr/ADR-7-inner-loop-tool-registry.md) §2 (ToolSpec); the v0.1 loader rejects `permission: "full"` and any tool missing a required field. Three-tier disclosure (server-name list → per-tool descriptors → full schema on demand) per ADR-7 §6. MCP-shaped request/response per [ADR-2 amendment 2026-05-01](../knowledge/adr/ADR-2-llm-tiering.md#amendment-2026-05-01--mcp-forward-compat-tool-shape-convention) (clarified 2026-05-12 — `error.code` accepts str or int). |
